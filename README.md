# Video generation with the Google Agent Development Kit (ADK)

This project creates a video from your prompt. The video can have any length. The default setting in this project is 24 seconds (3 concatenated scenes of 8 seconds each). The number of scenes is set by the `NUM_SCENES` constant.

It produces a video similar to `EXAMPLE_full_video.mp4`.

![Start image of scene 2](https://github.com/user-attachments/assets/30301e35-3bff-4572-a81e-810f95ee670f "Start image of scene 2")

It uses an agent that plans multi-scene explainers, generates reference frames, animates them with audio, and evaluates temporal consistency across scenes.

This project uses Google's "Veo" model for video generation. Veo is only available with prepaid billing and cannot be accessed with a free account. The cost for one execution of this project using the default setting of 3 scenes, which generates a video of 24 seconds duration, is approximately US$ 5 to US$ 10. The exact amount depends on the number of retries that are required to improve the quality. It will take approximately 5 to 10 minutes to run this project.

## How it works

It starts with a short and simple user prompt and ends up with a multi-scene explainer video with animated diagrams and synchronized audio. The agent uses tools to automate the process, from planning through generation to evaluation. Depending on the results, the execution can take different paths. For instance, if a scene doesn't pass evaluation, the agent can regenerate it with feedback rather than just moving on. It creates long videos while maintaining visual and audio consistency.

## Tools

The agent uses 4 tools. The first tool, `plan_scenes`, is only called once at the beginning. It creates the screenplay for all scenes.

The remaining 3 tools run in a fixed sequence loop of `generate_scene_image`, followed by `generate_scene_video`, followed by `evaluate_scene` (1 loop for each scene):

| Name                   |    Model    |                     Use |
| :--------------------- | :---------: | ----------------------: |
| `plan_scenes`          |   Gemini    | Break brief into scenes |
| `generate_scene_image` | Nano Banana |      Create start image |
| `generate_scene_video` |     Veo     | Create video with audio |
| `evaluate_scene`       |   Gemini    |           Score quality |

### First tool: `plan_scenes`

Input:

1. **Brief:** Subject, scope, target audience
2. **Num Scenes:** 3 (default, but can be changed)
3. **Voice Profile:** e.g. calm, male, neutral American

Note: The voice profile gets appended to every scene prompt to make the audio style consistent across the full video.

Output:

1. **`visual_description`:** Becomes the image generation prompt for the start image of the scene
2. **`narration_script`:** Approximately 20 words that fit in 8 seconds of spoken audio
3. **`camera_motion`:** Tells the video model (Veo) how to animate (zoom, pan, static)

### Second tool: `generate_scene_image`

Input:

1. `visual_description` (output from the `plan_scenes` tool)

Output:
An image that is used as the starting point of the scene

Note: A style prefix is appended to every image prompt to make sure all scenes maintain the same visual style.

### Third tool: `generate_scene_video`

Input:

1. **Reference Image:** The image generated by the second tool (`generate_scene_image`)
2. **Visual Prompt:** Camera motion, animation description
3. **Audio:** Narration script plus the voice profile

Output:
An 8 second video with audio (narration)

### Fourth tool: `evaluate_scene`

Input:

1. The video generated by the third tool (`generate_scene_video`) plus the original prompt

Output:
Scores (1-5 per criterion). The criteria here include temporal consistency, motion coherence, prompt adherence, and others. There is one key addition, which is the failure type. If the overall score is below the threshold, the evaluator classifies the failure as visual or audio. This classification is what tells the agent what to retry.

Note: This evaluation tool is very important, especially for long videos. Without it, the video might end up with scenes that feel disconnected from each other. The evaluation catches that and retries with specific feedback.

## Google ADK

Google's ADK is a powerful framework for building AI agents that can plan, reason, and use tools, with strong integration into Google's Gemini Large Language Models (LLMs) and cloud services. It is a strong choice for developers who want a polished, scalable, and Google-native way to build advanced agents. In the Google ADK, the instructions for the core model orchestrate the workflow, including decisions, loops, and handoffs.

## Google Cloud setup

This project uses Google AI (Google AI Studio). It can also easily be adapted to use Google Vertex AI (now evolved into Google's Gemini Enterprise Agent Platform).

To access all Google Gemini models for this example, you must:

1. [Create a Google account.](https://accounts.google.com/signup) When you are done, you will have a Google account that you can use for all Google services, including Google Cloud.
2. [Sign up for Google Cloud.](https://cloud.google.com/free) You must enter your credit card details, even just to get your free $ 300 trial credit for new customers. However, for this project, you must have paid billing enabled to use the video generation model "Veo".
3. Create your first project in Google Cloud.
4. Create an API key for your project.
5. Enter your Google API key in the `.env.example` file and rename this file to just `.env` (remove the ".example" ending).

## FFmpeg requirement

This project requires "FFmpeg" to join the individual 8 second video scenes together.

FFmpeg is a complete cross-platform solution to record, convert, and stream audio and video. You can install FFmpeg directly from the [FFmpeg project web site](https://www.ffmpeg.org/). If you are using Microsoft Windows, then you can just save the `ffmpeg.exe` file [that you can download from here](https://www.ffmpeg.org/download.html) into your project directory. On Linux, you might want to install a static build, if a package installation from the project website fails. You can download a static build [here](https://johnvansickle.com/ffmpeg/).

A missing installation of FFmpeg will usually trigger a strange error message like "FileNotFoundError: The system cannot find the file specified". This error message can be misleading, as it does not mean that it cannot find the input video file. It just means that it cannot find the FFmpeg installation.
